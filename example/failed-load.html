Saria’s random message loader program: Message loading failure

[this is the default content]

Explanation

This example shows what happens when the messages fail to load.

The target element is a <p> element: 

<p data-saria-random-message-src="nonexistent">[this is the default content]</p>

The url “nonexistent” does not exist, so the fetch attempt returns a “404 Not Found” error.

Because the messages were not found, the content in the element is not replaced.
