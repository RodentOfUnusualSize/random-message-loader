Saria’s random message loader program: HTTP preloading

Explanation

Note: This example requires configuration of the server to produce the proper HTTP response header field.

This page demonstrates using HTTP preload hints to load the script and/or messages file earlier.

In the HTTP response, the server should include Link fields, with the script and/or message source URLs.

For example, to preload the script, one might use: 

Link: <random-message-loader.js>; as=script; rel=preload

To preload the messages file, one might use: 

Link: <data/quotes.txt>; as=fetch; rel=preload; crossorigin=anonymous

In a supporting browser, depending on the technology being used, you may be able to observe the script and/or messages file being loaded concurrently with the page, or possibly even before the page begins to load.

Note: The actual method may be HTTP preload hints, HTTP/2 push, or 103 Early Hints, depending on the browser, server configuration, and exact format of the Link field data. The difference doesn’t really matter; the effect will be more or less the same.
