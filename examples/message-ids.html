Saria’s random message loader program: Message IDs

Explanation

This page demonstrates how a randomly-selected message can be repeated, via the use of message IDs.

The script is loaded just as in the basic example. The <figure> element used to receive the content is almost identical, except for the addition of the data-saria-random-message-id attribute: 

<figure data-saria-random-message-src="data/quotes.txt" data-saria-random-message-id="id"></figure>

The above element is repeated three times to show that the same message is repeated. (Compare this to the multiple messages example, where multiple elements without IDs each get different randomly-selected messages.)

The ID value can be anything (here it is just “id”). All elements that have the same source URL and the same ID will receive the same randomly-selected message. IDs are compared after leading and trailing whitespace is removed, but no other transformations are done (this means the comparison is case-sensitive: “foo” and “Foo” are two different IDs).

Empty message IDs are the same as no message IDs. Every element with an empty ID or no ID will get a different randomly-selected message.
